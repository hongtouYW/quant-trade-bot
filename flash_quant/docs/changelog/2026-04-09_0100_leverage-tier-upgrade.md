## 改动原因 (Why)

Hongtou 在审阅文档后提出:

## 团队讨论记录 (Party Mode)

### 📋 John (PM) 初始反应
- "这不是小改动, 是推翻 BR-001"
- 理解 Hongtou 的逻辑: 大币流动性好 → 可以加杠杆
- 但需要团队重新评估风险

### 🧪 Murat (TEA) 数学分析
**100x 杠杆的致命问题**:
- 100x + 300U 保证金 → 价格反向 0.5% = 爆仓 (150U 归零)
- BTC 5min 平均波幅 0.3-0.5% → 正常波动就能爆仓
- 止损 -10% ROI = 价格空间只有 0.1% → 被正常波动扫损概率 > 70%

**50x 的数学可行性**:
- 50x + 止损 -5% ROI = 价格空间 0.1% → 仍然紧, 但配合精准信号可行
- 假设胜率 70%: 期望值/笔 = 0.70 × 60U - 0.30 × 15U = 37.5U ✅ 正期望
- 假设胜率 60%: 期望值/笔 = 0.60 × 60U - 0.40 × 15U = 30U ✅ 仍正期望

**Murat 结论**: 50x BTC 数学可行, 100x 不可行 (需 > 80% 胜率, 目前做不到)

### 📊 Mary (Analyst) 数据支持
- BTC 5min K线影线统计: 影线 < 0.1% 的只有 ~30%
- 意味着 100x 止损 (0.01% 空间) 有 70% 概率先扫损再涨
- 建议按币种波动率分级杠杆

### 🏗️ Winston (Architect) 技术影响
- 代码改动: `MAX_LEVERAGE = 20` → `LEVERAGE_TIERS = {...}` 字典
- 50x+ 杠杆下, 下单延迟更致命 (30ms = 0.5% ROI)
- 必须用限价单, 市价单滑点被放大 50-100 倍
- 建议 `STOP_LIMIT` 替代 `STOP_MARKET` (更精确但极端行情可能不成交)

### 💻 Amelia (Dev) 实现担忧
- Binance `STOP_MARKET` 市价触发, 高杠杆滑点可能导致:
  - 设 -5% ROI 止损, 实际成交 -8% ROI (0.06% 滑点 × 50x)
- 解决: `STOP_LIMIT` + 本地兜底止损双保险

### 🧪 Murat 对 100x 的最终判断
**三人一致反对 100x**:
| 原因 | 数据 |
|---|---|
| 止损空间 0.01% 太窄 | BTC 正常影线 0.1-0.3% |
| 被正常波动扫损 | 70%+ 概率先亏后涨 |
| 滑点被放大 100 倍 | 0.05% 滑点 = 5% ROI |
| 下单延迟致命 | 30ms = 0.5% ROI |
| 数学不可行 | 需 > 80% 胜率 |

## 最终决策: 方案 A (分级杠杆)

### 新 BR-001 规则

| 币种分级 | 代表 | 杠杆 | 止损 ROI | 价格止损空间 | 单笔最大亏 |
|---|---|---|---|---|---|
| **Tier A Major** | BTC, ETH | **50x** | -5% | 0.1% | 15U |
| **Tier A Large** | SOL/BNB/XRP/DOGE/ADA/AVAX | **30x** | -8% | 0.27% | 24U |
| **Tier B 中盘** | Top 50 其他 | **20x** | -10% | 0.5% | 30U |

### Phase 4 渐进开放 100x 的条件
- Phase 3 胜率 ≥ 70% → 可试 **75x BTC** (小仓位)
- Phase 4 胜率 ≥ 75% → 可试 **100x BTC** (100U 保证金)

## 关键教训 (Lessons)

1. **高杠杆不等于高风险, 关键是止损空间** — 50x + 紧止损的单笔亏损 (15U) < 20x + 宽止损 (30U)
2. **100x 在当前信号精度下不可行** — 必须等信号精度提升到 75%+ 才考虑
3. **分级杠杆比统一杠杆更科学** — 不同币种波动率不同, 应区别对待
4. **团队讨论的价值** — 原始需求 "100x" 被数据驳回, 折中为 "50x + 渐进扩展"
