### 关键架构决策

**核心原则**:
- Boring Technology First (Flask/MySQL/Redis)
- 8 条 BR 硬编码,不在配置文件
- 单向 pipeline 数据流
- 状态全部持久化

**5 个关键技术决策**:
1. WebSocket 优先 vs REST (避开限速)
2. 三进程隔离 (web/engine/ws_collector)
3. 内存 K线缓存 + Redis CVD 缓存
4. SQLAlchemy Core 不用 ORM 关系映射
5. SSR + AJAX 不用 SPA

**24 个核心模块**:
- core/ (常量+异常+日志)
- data/ (K线缓存+CVD计算+市场数据+Redis)
- ws/ (WebSocket客户端+流处理)
- scanner/ (Tier 1/2/3 三个扫描器)
- filters/ (Wick/CVD/Funding/Blacklist)
- risk/ (单笔/连亏/时段/黑天鹅)
- executor/ (Paper/Binance)
- models/ (7 张表的 SQLAlchemy)
- web/ (routes + templates)
- config/ (settings + tier_params)

**7 张数据表**:
- signals (含过滤结果)
- trades
- positions
- daily_stats
- circuit_breaker
- audit_log
- (kline_cache 可选)

**8 条 BR 硬编码位置**:
| BR | 位置 |
|---|---|
| BR-001 杠杆≤20x | core/constants.py |
| BR-002 Tier D | filters/blacklist_filter.py |
| BR-003 新币 | filters/blacklist_filter.py |
| BR-004 渐进资金 | core/constants.py (枚举) |
| BR-005 单笔≤300U | core/constants.py |
| BR-006 服务端止损 | executor/binance_executor.py |
| BR-007 Tier 1 时段 | scanner/tier1_scalper.py |
| BR-008 周末减仓 | risk/position_risk.py |

**部署方案**:
- 服务器: Vultr Tokyo HFC-2c-4gb $24/月
- OS: Ubuntu 22.04 LTS
- 进程: 3 个 (web/engine/ws) 由 supervisord 管理
- 数据库: 独立 MySQL flash_quant_db
- 反代: Nginx + Let's Encrypt SSL
