# Flash Quant — 信号不足解决方案

> **日期**: 2026-04-11
> **问题**: Tier 1 阈值 (量比≥5x + 5min涨幅≥2%) 在正常行情几乎不触发,Dashboard 无数据

---

## 📊 当前数据

| 指标 | 现状 |
|---|---|
| BTC 5min 平均涨跌幅 | 0.06% ~ 0.15% |
| Tier 1 要求 | ≥ 2% (差 **13 倍**) |
| 引擎运行时间 | 24+ 小时 |
| 有效信号 | 仅 WLDUSDT (1次) + CFXUSDT (1次) + ARBUSDT (1次) |
| PRD 预估 | 日均 10-20 信号 |
| 实际 | 日均 ~1-3 信号 |

**结论**: Tier 1 是"极端行情捕捉器", 不适合当唯一信号源。PRD 本来就设计了三层,现在需要实现 Tier 2 和 Tier 3。

---

## 📋 方案 A: 实现 Tier 2 趋势扫描器 (推荐)

**来源**: PRD FR-002

### 触发条件 (按 PRD)
```
15min MACD 柱状图首次转色 (变正/变负)
+ 15min RSI 突破 65 (多) 或跌破 35 (空)
+ 15min EMA9 上穿/下穿 EMA21
+ 1h 成交量 > 24h 均量 × 1.5
+ Wick + Funding 过滤 (CVD Phase 1 跳过)
```

### 参数
| 项 | 值 |
|---|---|
| 扫描周期 | 60 秒 |
| K线周期 | 15min |
| 监控范围 | 50 个币 (同 Tier 1) |
| 杠杆 | 按分级: BTC 50x / SOL 30x / 中盘 20x |
| 止损 | 按分级: -5% / -8% / -10% ROI |
| 止盈 | trailing stop (6% 启动, 3% 回撤) |
| 最大持仓 | 2-8 小时 |
| 时段 | 全天 (不受 UTC 8-22 限制) |

### 预期信号量
```
MACD 转色:        日均 ~30 次/币 → 50 币 = ~1500
+ RSI 突破:       砍掉 ~80% → ~300
+ EMA 穿越:       砍掉 ~70% → ~90
+ Volume 过滤:    砍掉 ~50% → ~45
+ Wick 过滤:      砍掉 ~30% → ~30
最终: 日均 ~20-30 信号 ✅
```

### 工作量
- 新建 `scanner/tier2_trend.py` (~200 行)
- 需要 TA-Lib (服务器已有 ccxt, 需要装 ta-lib 或用纯 Python 计算)
- 预计 2-3 小时完成

### 优点
- ✅ 严格按 PRD FR-002 实现
- ✅ 日均 20-30 信号, 满足 AC-2 要求
- ✅ 正常行情也能触发 (不需要极端波动)
- ✅ MACD/RSI/EMA 是成熟的技术指标

### 缺点
- ⚠️ 需要 15min K线缓存至少 50 根 (warmup ~12.5 小时)
- ⚠️ 信号可能偏多, 需要调参

## 📋 方案 B: 实现 Tier 3 方向扫描器

**来源**: PRD FR-003

### 触发条件 (按 PRD)
```
1H 整点扫描
评分 = RSI(25) + MA(25) + Volume(25) + Position(25) + MACD(10) + ADX(10) + BB(5)
评分 ≥ 75 → 触发
```

### 预期信号量
```
评分 70-79: 日均 ~8
评分 80+:   日均 ~3
总计: 日均 ~5-10 信号
```

### 工作量
- 新建 `scanner/tier3_direction.py` (~300 行, 评分系统复杂)
- 预计 3-4 小时

### 优点
- ✅ 评分系统全面
- ✅ 1H 周期更稳

### 缺点
- ⚠️ 信号较少 (日均 5-10)
- ⚠️ 实现复杂 (7 个指标评分)
- ⚠️ 需要 1H K线缓存 warmup (~24 小时)

## 📋 方案 C: 降低 Tier 1 阈值

**不改 PRD, 只调参数**

### 调整
| 参数 | 原值 | 调整值 |
|---|---|---|
| 量比 | ≥ 5x | ≥ 3x |
| 涨幅 | ≥ 2% | ≥ 0.8% |

### 预期
```
量比 ≥ 3x + 涨幅 ≥ 0.8%:
  BTC: 日均 ~3-5 信号
  全部 50 币: 日均 ~30-50 信号
  + Wick 过滤: ~15-25
```

### 优点
- ✅ 最快 (改 2 个数字, 5 分钟搞定)
- ✅ 立刻有数据

### 缺点
- ⚠️ 偏离 PRD 原始设计 (PRD 说 5x/2%)
- ⚠️ 信号质量下降 (量比 3x 假信号更多)
- ⚠️ 不是根本解决办法

---

## 📋 方案 D: A + C 同时做 (最佳)

```
1. 先做方案 C (5 分钟) → 立刻有信号, Dashboard 有数据看
2. 再做方案 A (2-3 小时) → Tier 2 上线, 正式信号源
3. Tier 1 阈值恢复正式值 (5x/2%) → 只在极端行情触发
4. 后续做方案 B → Tier 3 补充
```

**最终效果**:
```
Tier 1 (极速): 极端行情 → 日均 1-3 信号 (稀少但精准)
Tier 2 (趋势): 正常行情 → 日均 20-30 信号 (主力信号源)
Tier 3 (方向): 补充中线 → 日均 5-10 信号 (后续加)
合计: 日均 25-40 信号
```
